# CLAUDE.md

Guidance for Claude Code (and any agent) working in this repository.

## What this repo is

A **collection of reusable agents and skills** for two platforms — [Claude Code](https://docs.claude.com/en/docs/claude-code) and [opencode](https://opencode.ai). It is a browsable, shareable catalog, not an application. There is no build or test step.

## Layout

```
claude/      # Claude Code items — the catalog (source of truth)
  agents/<name>.md
  skills/<name>/SKILL.md
opencode/    # opencode items — the catalog (source of truth)
  agents/<name>.md
  skills/<name>/SKILL.md
scripts/link.sh   # wires catalog items into discovery dirs
```

- `.claude/` and `.opencode/` are **generated by `scripts/link.sh` and git-ignored**. They contain only symlinks back into the catalog. Never commit them; never edit an item through them — edit the real file under `claude/` or `opencode/`.
- Docs: `README.md` (catalog + how to run/install), `CONCEPTS.md` (what agents/skills are + best practices), `CONTRIBUTING.md` (how to add an item), plus a per-platform-per-type `README.md` guide in each folder.

## Adding or changing an item

1. Put the real file under `claude/<type>/` or `opencode/<type>/`. Names are **kebab-case** and kept **identical across platforms** when an item ships for both.
2. Run `./scripts/link.sh` to (re)generate the local discovery symlinks so it runs in-repo.
3. Update **both** the catalog table in `README.md` **and** the "Items in this collection" table in the relevant folder guide.
4. Match the surrounding style; keep descriptions to one line.

## Format rules that are easy to get wrong

- **Claude Code agent** (`claude/agents/<name>.md`): YAML frontmatter, only `name` + `description` required; `tools` is a comma-separated allowlist (omit to inherit all); `model` is `sonnet`/`opus`/`haiku`/`fable`/full-id/`inherit` (defaults to `inherit`).
- **Claude Code skill** (`claude/skills/<name>/SKILL.md`): the folder name becomes the `/command`; `description` should be a precise trigger.
- **opencode agent** (`opencode/agents/<name>.md`): frontmatter `description` required, `mode` (`primary`/`subagent`/`all`), `tools` is a **map of tool→boolean** (not a list), `model` is `provider/model`. The **filename** is the agent name.
- **opencode skill** (`opencode/skills/<name>/SKILL.md`): same `SKILL.md` standard; agent-invoked via the `skill` tool (no `/slash`).
- **opencode directories are plural** (`agents/`, `skills/`) — canonical. Singular (`agent/`, `skill/`) only works for backwards compatibility; do not introduce it.
